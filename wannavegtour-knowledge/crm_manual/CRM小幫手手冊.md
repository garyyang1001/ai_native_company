## LINE

### MAN-LINE-01 登入 CRM 後台(只有 LINE 登入,沒有帳號密碼)(v1)
*頁面:CRM 後台網址 → /login 登入頁 → 「使用 LINE 登入」*

CRM 沒有帳號密碼這回事,全系統只用 LINE 登入:
1. 打開 CRM 後台網址,會自動被導到登入頁(/login)。
2. 頁面上只有一顆綠色「使用 LINE 登入」按鈕,上方提示「請使用 LINE 帳號登入」。
3. 按下去會跳到 LINE 的授權畫面,用你自己的 LINE 帳號同意授權即可進入後台。
4. 第一次登入系統會自動幫你建一個「唯讀(readonly)」身分的使用者;要拿到管理權限,需要由現有管理員從「Line 帳號管理 → 成員管理」發邀請連結給你(見 MAN-LINE-06)。

注意:沒有「忘記密碼」流程,也不需要請人開帳號密碼——只要你的 LINE 登得進去就能登入。

### MAN-LINE-02 打開「Line 帳號管理」的兩條路(聊天頁齒輪/帳號下拉選單)(v1)
*頁面:聊天頁 → 設定(齒輪) → 左側「Line 帳號管理」;或 聊天頁左上帳號下拉 → 「管理帳號...」*

「Line 帳號管理」藏在聊天頁的設定視窗裡,有兩條路:

路徑一(齒輪):
1. 進入聊天頁。
2. 點設定按鈕(齒輪圖示)——桌面版在左側窄邊欄最下方,聊天室列表右上角也有一顆齒輪。
3. 跳出設定視窗,左側選單點「Line 帳號管理」。

路徑二(捷徑):
1. 聊天頁左上的 LINE 帳號切換器(顯示目前官方帳號名稱的那塊)點開。
2. 下拉選單最底下選「管理帳號...」,會直接打開設定視窗並停在「Line 帳號管理」。

進去後標題是「Line 官方帳號管理」,副標「管理您的 Line Bot 憑證 (Channel Secret / Token)」,每個官方帳號一張卡片。設定視窗左側目前看得到的分頁只有:資訊欄位設定、標籤管理、Line 帳號管理、管理員設定、流程通知設定(其餘如罐頭訊息、群組管理等已暫時隱藏)。

### MAN-LINE-03 新增一個 LINE 官方帳號到 CRM(v1)
*頁面:聊天頁 → 設定(齒輪) → Line 帳號管理 → 「新增帳號」*

1. 打開「Line 帳號管理」(見 MAN-LINE-02)。
2. 點右上角「新增帳號」按鈕。系統會先檢查方案配額,額滿會跳出「無法新增帳號」並顯示目前用量(例如 2/2)——這時要請管理員升級方案,不是操作錯誤。
3. 在「新增 Line 官方帳號」視窗填:
   - 帳號名稱(必填,例如:客服機器人)
   - 大頭貼(選填,可貼網址或點「上傳圖片 (Max 5MB)」)
   - Channel ID(選填)
   - Channel Secret(必填)
   - Channel Access Token (Long-lived)(必填)
   Secret 和 Token 要去 LINE Developers 後台該 channel 的設定頁複製。
4. 點「建立帳號」,成功會顯示「帳號建立成功」,列表多一張卡片。
5. 建好後務必接著設定 Webhook URL(見 MAN-LINE-04),不然收不到客人訊息。

### MAN-LINE-04 把 Webhook URL 填回 LINE Developers(讓客人訊息進得來)(v1)
*頁面:聊天頁 → 設定(齒輪) → Line 帳號管理 → 帳號卡片 → 「Webhook URL」區塊(複製) → 貼到 LINE Developers 的 Messaging API > Webhook settings*

每個 LINE 官方帳號在 CRM 都有一條專屬的 Webhook URL,要回填到 LINE 官方後台才會通:
1. 打開「Line 帳號管理」,找到該帳號的卡片。
2. 卡片下方灰色區塊「Webhook URL」就是那條網址,格式固定是 https://crm-api.lazyoffice.app/api/v1/line/callback/<每個帳號專屬的字尾>,旁邊有複製按鈕可直接複製。
3. 卡片上也有提示文字:「請將此網址填入 Line Developers 後台的 Messaging API > Webhook settings」——照做即可:到 LINE Developers 該 channel → Messaging API 分頁 → Webhook settings 貼上,並把 Webhook 開關打開。

注意:這條網址永遠是正式機網域(crm-api.lazyoffice.app),不要自己改成別的網址或 localhost。

### MAN-LINE-05 修改 LINE 帳號的名稱、大頭貼或憑證(Secret/Token)(v1)
*頁面:聊天頁 → 設定(齒輪) → Line 帳號管理 → 帳號卡片 → 鉛筆(編輯)圖示*

1. 打開「Line 帳號管理」,在要改的帳號卡片底部點鉛筆(編輯)圖示。
2. 卡片會切成編輯表單,可改:
   - 名稱(必填)
   - 大頭貼(貼網址或「上傳圖片 (Max 5MB)」,下方會即時預覽)
   - 新 Channel Secret (留空則不修改)
   - 新 Access Token (留空則不修改)
3. 重點:Secret 和 Token 欄位「留空就是不動舊值」,所以只想改名字時不用擔心把憑證洗掉;反之要換新憑證就把新值貼進去。
4. 點「儲存」完成,或「取消」放棄。

卡片檢視模式也會顯示帳號狀態(運作中/停用)與你在這個帳號的角色(OWNER/MEMBER 標籤)。

### MAN-LINE-06 邀請同事成為 LINE 帳號管理員(一次性、5 分鐘有效連結)(v1)
*頁面:聊天頁 → 設定(齒輪) → Line 帳號管理 → 帳號卡片「成員管理」圖示 → 「邀請管理員」→ 複製連結*

邀請是在 CRM 裡操作,不是去 LINE Official Account Manager:
1. 打開「Line 帳號管理」(聊天頁齒輪 → Line 帳號管理)。
2. 在要授權的帳號卡片底部,點「成員管理」圖示(加人的小圖示,鉛筆和垃圾桶中間那顆)。
3. 跳出「成員管理」視窗,可看到現有管理員名單(使用者、角色 OWNER/ADMIN、加入時間)。
4. 點右上「邀請管理員」按鈕,系統生成一條邀請連結,並提示「邀請連結已生成 (5分鐘有效)」。
5. 連結會顯示在「邀請連結 (一次性 + 5分鐘有效)」的藍色提示框裡,按「複製」傳給同事(LINE、訊息都可以)。

兩條鐵則:連結「只能被點用一次」、「5 分鐘後過期」。所以複製完要馬上傳,並請同事馬上點開。和 Email 完全無關——系統用 LINE 登入,沒有「邀請綁 Email」這種機制。

### MAN-LINE-07 同事收到邀請連結後怎麼接受?顯示無效/過期怎麼辦?(v1)
*頁面:(同事端)邀請連結 /invite/<代碼> → 前往登入 / 註冊(LINE 登入) → 接受邀請頁 → 「確認加入」*

同事端的流程(連結長相是 /invite/<一串代碼>):
1. 點開連結會看到「管理員邀請」頁,顯示是哪個 LINE 官方帳號邀請、要給什麼角色。
2. 如果還沒登入,頁面會提示「請先登入:您需要登入 Line 帳號並連結此系統才能接受邀請」,點「前往登入 / 註冊」用 LINE 登入後再回到連結。
3. 登入後頁面變成「接受邀請」,確認帳號與身份無誤,點「確認加入」,看到「已成功加入!」就完成了。

看到「無效的邀請」或「此邀請連結已過期」時:
- 原因只有兩種:超過 5 分鐘,或連結已被點用過(一次性)。
- 解法:請原本發邀請的人回「Line 帳號管理 → 成員管理 → 邀請管理員」重新生成一條新連結,同事在 5 分鐘內點開、先完成 LINE 登入、再按「確認加入」。
- 不要去檢查 Email——邀請和 Email 完全無關。

### MAN-LINE-08 刪除 LINE 官方帳號(要輸入 LINE 收到的 6 位數驗證碼)(v1)
*頁面:聊天頁 → 設定(齒輪) → Line 帳號管理 → 帳號卡片紅色垃圾桶圖示 → LINE 收 6 位數驗證碼 → 「確認刪除帳號」視窗輸入 → 刪除*

刪除有安全驗證,不是按一下就刪:
1. 打開「Line 帳號管理」,在帳號卡片底部點紅色垃圾桶(刪除)圖示。
2. 瀏覽器先跳確認框:「確定要申請刪除帳號 "某某" 嗎?系統將發送驗證碼至您的 Line 帳號。」按確定。
3. 系統會把 6 位數驗證碼用 LINE 訊息發給「你本人」的 LINE,畫面顯示「驗證碼已發送,請檢查您的 Line 訊息」。
4. 跳出「確認刪除帳號」視窗,把 LINE 收到的 6 位數驗證碼輸入後按「刪除」。
5. 驗證成功後系統會「在背景排程刪除」——該帳號的相關對話紀錄會被陸續刪掉,而且無法復原,刪之前務必確認。

如果一直收不到驗證碼,先確認自己的 LINE 有加該官方帳號好友、且自己的 CRM 帳號有綁定 LINE。

### MAN-LINE-09 LIFF 客人端頁面總覽:客人在 LINE 裡填的那些表單是什麼?(v1)
*頁面:(客人端)LINE 聊天室收到的連結 → LIFF 頁面;(同仁端)側欄「團體行程」→ 該團詳細頁 → 通知按鈕(催繳訂金/催繳尾款/催填旅客資料/催繳護照)*

客人不會登入 CRM 後台;他們是在 LINE 裡打開我們推送的 LIFF 頁面(手機裡長得像綠色主題的網頁表單),填完直接寫進 CRM。共有這幾頁:
- /liff/traveler-form「旅客資料填寫」:填同行旅客姓名、證件號等,送出後寫入旅客名單,客人會收到完成通知。
- /liff/passport-form:上傳護照/台胞證照片(有 OCR 自動辨識護照號碼,OCR 目前僅支援護照)。
- /liff/my-orders「我的訂單」:客人查自己的訂單。
- /liff/payment-upload「付款上傳」:客人上傳匯款截圖,後台會即時收到通知。
- /liff/payment-gateway「付款」:線上刷卡/付款頁。
- /liff/trip-confirm「行程確認」:客人按下確認參加,訂單狀態會更新並回推 LINE 訊息。

同仁端怎麼觸發?最常用的是「團體行程」詳細頁的通知功能,有四種:催繳訂金、催繳尾款、催填旅客資料、催繳護照——系統會自動發 LINE 通知(含對應的 LIFF 連結)給所有已綁定 LINE 的客戶;發送前會先確認(例如訂金通知會擋下尚未完成配房的訂單,只發給已完成配房者)。

注意:這些頁面送出都是「動真格」——會寫進正式資料庫、還會回推 LINE 訊息給客人,不要拿真客人的連結做測試。

### MAN-LINE-10 想看客人有沒有綁定 LINE,去哪裡看?(v1)
*頁面:側欄「客戶管理」→ 客戶列表(綠點/灰點、line 篩選) → 客戶詳細頁*

看「客戶管理」頁最快:
1. 側欄點「客戶管理」進客戶列表(/customers),頁面副標就是「管理所有客戶資料與 LINE 綁定狀態」。
2. 每位客人名字旁有一個小圓點:綠點=已綁定 LINE,灰點=未綁定。
3. 上方統計卡會顯示已綁定 LINE 的人數;篩選器裡有「line」選項,可以只列出已綁定的客人。
4. 點進單一客人的詳細頁(/customers/:id)可看更完整的 LINE 綁定資訊(含綁定日期)。

客人還沒綁怎麼辦?同仁是可以手動綁的:在客戶編輯表單裡有 LINE User ID 欄位可直接填入(後端也有對應的綁定 API),不是只能等客人自己完成綁定流程。沒綁定的客人收不到催繳/通知類的 LINE 推播(見 MAN-LINE-09),所以發通知前先確認綁定狀態。

## 催繳

### MAN-催繳-01 怎麼進入「繳款提醒」頁面?誰能進?(v1)
*頁面:直接在網址列輸入 /payment-reminders(需先登入;側欄會亮在「付款管理」(PAYMENTS),但側欄/首頁目前都沒有直接連結)*

1. 繳款提醒的頁面路徑是 /payment-reminders,頁面標題是「繳款提醒」;系統把它歸在左側選單「付款管理」(PAYMENTS) 底下(進到這頁時,側欄會亮在「付款管理」)。
2. 注意:側欄的「付款管理」按下去是進「付款管理」(/payments) 頁,側欄上沒有「繳款提醒」的直接連結;首頁的「繳款提醒」捷徑目前被暫時隱藏(程式碼註解掉了);整個前端也沒有其他按鈕連過去。所以目前唯一的進法是直接在瀏覽器網址列輸入 /payment-reminders。
3. 權限:要先登入 CRM 後台(用 LINE 登入)才能看,沒登入會被導回登入頁。登入之後就能進——程式碼上只檢查「有沒有登入」,沒有再分管理員或一般帳號。(以前流傳「僅管理員可進」,程式碼裡沒有這個限制。)
4. 沒有叫「催繳管理」的選單——以前小幫手講過這個名字,是錯的,不要去找。

### MAN-催繳-02 幫一筆訂單新增催繳提醒(欄位怎麼填)(v1)
*頁面:繳款提醒列表 (/payment-reminders) → 右上「+ 新增提醒」→ 新增繳款提醒 (/payment-reminders/new)*

1. 進「繳款提醒」列表頁 (/payment-reminders),按右上角藍色「+ 新增提醒」,進入「新增繳款提醒」頁 (/payment-reminders/new)。
2. 必填三項(沒填會跳「提醒編號、訂單和截止日期為必填」):
   - 提醒編號:自己編一個,例如 REM-2024-001。
   - 訂單:用下拉選(顯示「訂單編號 - 客戶姓名」),選好後「訂單編號」「客戶姓名」會自動帶入(新增時「訂單編號」是唯讀的,不用自己打)。
   - 截止日期:選日期後下方會即時顯示「已逾期 X 天」(紅字)、「今天到期!」或「X 天後到期」。
3. 其他欄位:
   - 提醒類型:定金 / 尾款(預設定金)。
   - 提醒狀態:待處理 / 已完成 / 已逾期 / 已取消(新建就留「待處理」)。
   - 提醒次數:預設 0,不用動。
   - 付款狀態:未繳 / 已繳 / 免繳(新建通常留「未繳」)。
   - 備註:選填。
4. 按「新增提醒」儲存,會跳回列表頁。注意:列表頁本身顯示有異常(見 MAN-催繳-03),跳回去看到空白或怪資料不代表沒建成功。
5. 注意:沒有「應繳金額」這種欄位,金額不在提醒表單上;建好提醒系統也不會自動發送,發送都要人工處理(見 MAN-催繳-04)。

### MAN-催繳-03 繳款提醒列表頁怎麼看?(此頁顯示目前異常)(v1)
*頁面:繳款提醒列表 (/payment-reminders)*

1. 列表表格設計上有六欄:訂單 / 類型 / 截止日期 / 金額 / 狀態 / 操作。狀態設計上分:待處理(黃)、已發送(藍)、已繳清(綠)、已取消(灰)。
2. 【此功能目前異常】列表程式讀的欄位名稱和後端送回來的資料對不上(程式讀 due_date / amount_due / status,但後端實際欄位是 deadline_date / reminder_status / payment_status,而且後端根本沒有 amount_due 金額欄位):
   - 「截止日期」會顯示成無效日期。
   - 「金額」欄讀不到資料會出錯——只要列表裡有任何一筆提醒,整頁幾乎一定當掉、顯示空白。
   - 「狀態」標籤顯示不出正確狀態。
   - 「訂單」欄會顯示一串訂單 ID 而不是訂單編號。
3. 替代作法:因為整頁可能空白,列表上的「編輯」按鈕也可能按不到。要確認某筆提醒的正確內容,可以直接在網址列輸入 /payment-reminders/{該筆ID}/edit 進編輯頁看(編輯頁欄位是正確的;ID 請管理員從後端查),或整筆都請管理員從後端 API 查。
4. 此頁修好前,請不要依列表上顯示的日期/金額/狀態做判斷。

### MAN-催繳-04 怎麼把催繳提醒發送給客人?(發送功能目前異常)(v1)
*頁面:繳款提醒列表 (/payment-reminders) → 操作欄「發送」(目前按鈕不會出現,需管理員走後端 API)*

1. 先講重點:系統不會自動發送催繳訊息,所有發送都是人工觸發。
2. 設計上,列表「操作」欄在提醒是「待處理」時會出現「發送」按鈕,按了成功會跳「提醒已成功發送」。
3. 【此功能目前異常】兩層問題:
   - 因為列表欄位名稱不一致(見 MAN-催繳-03),列表頁本身可能整頁空白,而且「發送」按鈕的顯示條件永遠不成立,按鈕根本不會出現。
   - 就算出現,前端呼叫的 API 路徑(POST .../send)和後端實際路徑(PUT .../send-reminder)對不上,也會失敗。
4. 後端的發送 API(PUT /api/v1/payment-reminders/{id}/send-reminder)在 2026-06-10 已修復並通過測試:會依提醒類型發 LINE(定金=文字訊息、尾款=Flex 卡片);就算 LINE 發送失敗也不會報錯,仍會把提醒次數 +1 並記錄發送時間。
5. 但注意:修復時只驗證了「發送失敗時系統行為正確」,「真的把訊息送到客人 LINE」這條成功路徑還沒實測過——尤其定金文字提醒在修復前從來沒成功發出過。
6. 實務作法:需要發送催繳時,請找管理員從後端 API 操作,並先用測試帳號驗證一次訊息真的送得出去,再對真客人發。

### MAN-催繳-08 要作廢「繳費連結」不是在繳款提醒——在「金流訂單」(v1)

### MAN-分房-04 幫客人配房(設定配房對象)(v1)

### MAN-分房-05 房型偏好有哪些選項?補差金額怎麼填?(v1)
*頁面:/room-allocations/new 或 /room-allocations/{id}/edit → 「房型偏好」下拉 + 「補差金額」欄位*

在「新增/編輯房型分配」表單裡,「房型偏好」下拉共有 8 個選項(欄位下方註明「配房需求說明」):
1. 補房差(單人房)— 客人自己住一間,要補單人房差價。
2. 男生配房 — 男生彼此湊房。
3. 女生配房 — 女生彼此湊房。
4. 一大床。
5. 三人一房(加床不加價)。
6. 小朋友佔床(不加價)。
7. 由工作人員配房 — 客人沒意見,我們安排。
8. 不需要。
(不選就維持「未指定」。)
補差金額:選了「補房差(單人房)」這類要加錢的情況,把金額填在「補差金額」欄(數字,最小 0,可到小數兩位),然後按「更新分配」或「新增分配」儲存。

### MAN-分房-07 查看客人送來的換房申請(分房變更審核)(v1)

步驟:
1. 按側欄「分房審核」(或首頁的「分房變更審核」捷徑卡),進「分房變更審核」頁(/room-changes)。
2. 右上角兩個篩選鈕:「待審核」(預設)和「全部」。

4. 點卡片任意處展開明細:
   - 逐位旅客的差異表:欄位/變更前(紅色刪除線)/變更後(綠色),涵蓋姓名、房型(單人房/雙人房/三人房/四人房/三人一房(加床))、房號、不佔床。

### MAN-分房-08 核准或駁回換房申請(會 LINE 通知客人)(v1)
*頁面:/room-changes → 待審核卡片展開 → 「核准」/「駁回」→(駁回需填原因)→「確認駁回」*

只有「待審核」狀態的申請,展開後底部才有兩顆按鈕:
【核准】
1. 點綠色「核准」,會跳出「確定要核准此分房變更嗎?」確認框,按確定送出。
2. 核准後系統會更新訂單的旅客資料、重算加價購,並【自動發 LINE 訊息通知客人】(內容是「您的分房變更已核准」)。
【駁回】
1. 點白底紅框「駁回」,跳出「駁回分房變更」視窗。
2. 視窗有黃色警告:「以下駁回原因將透過 LINE 訊息通知客戶,請填寫清楚易懂的說明。」
3. 駁回原因【必填】(沒填字「確認駁回」按鈕按不下去),例如:「此房型已滿,請重新選擇」。
4. 按紅色「確認駁回」送出;按「取消」放棄。駁回後客人會收到含你填的原因的 LINE 訊息。
提醒:核准和駁回都會直接推 LINE 給客人,送出前請再次確認內容與用字;審核完成後卡片會記錄審核者與時間。

### MAN-團體-02 我要開一個新團,去哪裡建?哪些欄位必填?(v1)
*頁面:直接輸入網址 /tour-groups/new(列表頁無新增按鈕)→「新增團體行程」表單*

1. 【重要】團體行程列表頁目前「沒有」新增按鈕——這不是你眼花,程式碼裡確實沒做這顆按鈕(已驗證)。要建新團,請直接在瀏覽器網址列輸入:CRM網址/tour-groups/new,就會進到「新增團體行程」表單。

### MAN-團體-04 「缺男」「缺女」是什麼意思?系統怎麼算的?(v1)
*頁面:側欄「團體行程」→ /tour-groups 列表,行程名稱下方紅色標籤 +「只顯示缺男缺女」勾選框*

1. 意思:這一團「還沒分房」的旅客裡,同性別人數是奇數,代表會有一個人落單湊不成房,需要再收一位同性別的客人(或調整分房)。
2. 計算規則(由系統自動算,不用手動設定):
   - 只看「未取消訂單」裡「尚未配房」(沒有房號)的旅客。
   - 未配房的男生是奇數 → 顯示「缺男」;未配房的女生是奇數 → 顯示「缺女」;可能同時出現。
   - 旅客資料裡性別沒填(或填了男女以外的值)的人不會被計入。
3. 在哪裡看:
   - 列表頁行程名稱下方的紅色標籤「缺男・缺女」,滑鼠移上去會顯示「未分房:男 X / 女 Y」。
   - 用篩選列的「只顯示缺男缺女」勾選框可以只看有缺口的團。
4. 【目前狀況注意】2026-06-10 系統稽核發現,資料庫的旅客(passengers)資料目前是 0 筆(訂單有上千筆但旅客名單是空的,舊備份裡有 51 位旅客,疑似搬家時資料缺口,待跟 Gary 確認)。在旅客資料補回來之前,所有團都不會顯示缺男缺女標籤——看不到不代表沒有缺口,請先以人工確認為準。

### MAN-團體-06 想看某一團還有誰沒交護照/證件,要去哪裡看?(v1)

   - 每位旅客右側有「新增護照」或「編輯護照」按鈕,可以幫客人手動補登或修正護照資料。
4. 【目前狀況注意】2026-06-10 稽核發現本機資料庫的旅客與護照資料為 0 筆(舊備份裡有 51 位旅客/9 本護照,疑似搬家資料缺口,待跟 Gary 確認)。資料補回前,這個分頁可能顯示「尚無訂單」或進度全為 0,不代表客人真的都沒交。

### MAN-團體-08 怎麼把團切到「收訂金」「收尾款」?切換會發生什麼事?(v1)
*頁面:團體行程詳細頁(/tour-groups/:id)→「流程狀態」進度條 → 確認視窗「確認切換」*

1. 位置:該團詳細頁上方的「流程狀態」進度條,三步驟:1 收資料 → 2 收訂金 → 3 收尾款。點想切換的步驟(只能點相鄰一步或往回,不能跳兩步)。
2. 按下後會跳出確認視窗(切收訂金時視窗標題寫「切換到『收定金』」,「收定金」=「收訂金」,同一件事),提醒你:「系統將自動發送通知給所有已綁定 LINE 的客戶」。可在「自訂通知訊息(選填)」欄加一段話(例:繳費期限為 3/20,逾期將取消保留席位),再按「確認切換」才會真的切換並發通知;按「取消」則不動作。
3. 切到「收訂金」時,系統會先檢查配房狀況:還沒完成配房的訂單會列在橘色警告框裡(顯示每筆已配房 X/Y 人),這些訂單「不會」收到訂金通知,只有配房完成的才會收到——要全員都收到通知,請先去「分房管理」分頁把房配完。
4. 另外在編輯表單那邊也有保護:若行程的流程狀態是收訂金/收尾款,對應的繳款截止日不能早於今天;截止日距今 3 天內(含)會再跳一次確認視窗。
5. 小提醒:切換動作會自動發 LINE 通知給客人(後端 flow-status API 內建自動推播,已稽核確認),確定文案沒問題再按確認。

### MAN-團體-09 建好的團要改資料(名額、價格、截止日)去哪改?(v1)

1. 路徑:團體行程列表 → 點進該團詳細頁 → 右上角藍色「編輯」按鈕,進入「編輯團體行程」表單(/tour-groups/:id/edit)。

3. 「其他設定」兩個勾選:
   - 「搜集旅客資料」勾選後,客人可透過 LINE 的訂單卡片自己填旅客資料。
   - 「停止更新旅客資訊」勾選後,LINE 客人就不能再修改旅客資訊(通常開票前鎖定名單用),勾選時會出現橘色警告提示。
4. 改完按紫色「更新團體行程」儲存,會回到該團詳細頁;「取消」放棄修改。
5. 注意:若流程狀態是收訂金/收尾款,儲存時系統會檢查對應截止日不能早於今天,3 天內(含)到期會跳確認視窗(見 MAN-團體-08 第 4 點)。

### MAN-客戶-01 客戶列表在哪裡?怎麼找一位客人?(v1)

1. 左側選單點「客戶管理」(CUSTOMERS),會進到客戶列表頁(網址 /customers),頁面副標寫著「管理所有客戶資料與 LINE 綁定狀態」。

3. 搜尋框右邊還有「重置」按鈕,一鍵清掉搜尋字和篩選條件、回到第一頁。

### MAN-客戶-02 怎麼看客人有沒有綁定 LINE?(綠點/灰點)(v1)
*頁面:左側選單「客戶管理」→ 客戶列表 LINE 欄綠/灰點;點「已綁 LINE」統計卡篩選(僅當頁);點進客戶詳細頁看「LINE 綁定」與「LINE User ID」*

1. 客戶列表(/customers)的「LINE」欄位用小圓點表示綁定狀態:綠點=已綁定 LINE,灰點=未綁定(滑鼠移上去會顯示文字提示)。
2. 列表上方有四張統計卡:「總客戶 / VIP 客戶 / 已綁 LINE / 黑名單」。點「已綁 LINE」那張卡,列表就只剩已綁定的客人;點「總客戶」恢復全部。
3. 【範圍提醒】統計卡的 VIP/已綁 LINE/黑名單「數字」和「篩選結果」都只算「目前這一頁 50 筆」,不是全部客戶;要看全部請逐頁翻或以「總客戶」數為準。

5. 詳細頁還有「LINE User ID」欄,已綁定會顯示一串代碼,旁邊有複製按鈕可以一鍵複製。
6. 客人名字底下若有灰色小字,那是他的 LINE 顯示名稱(暱稱)。

### MAN-客戶-03 怎麼新增一位客戶?(v1)

### MAN-客戶-04 怎麼修改客戶資料?客人沒綁 LINE 可以幫他手動綁嗎?(v1)
*頁面:客戶列表「編輯」鉛筆 或 客戶詳細頁「編輯」→ 編輯客戶表單 → 改「LINE 用戶 ID」等欄位 → 「更新客戶」*

1. 改客戶資料有兩個入口:客戶列表該列的鉛筆「編輯」按鈕,或客戶詳細頁右上角的「編輯」按鈕,都會進到編輯客戶表單(/customers/客戶編號/edit)。
2. 表單可以改:姓名、電話(這兩個必填)、電子郵件、LINE 用戶 ID、標籤、備註。改完按「更新客戶」儲存。
3. 手動綁 LINE:可以。編輯表單裡的「LINE 用戶 ID」欄位是可以直接填寫的,把客人的 LINE User ID(U 開頭的一串英數字)貼進去後儲存,該客人就會變成已綁定(列表上變綠點)。後端也有對應的綁定 API(POST /customers/{id}/line-bind)。
4. 提醒:後台沒有現成的「查某位客人 LINE User ID」工具;如果手上沒有客人的 ID,讓客人自己走 LINE 端流程綁定比較保險,拿到正確 ID 再手動填。
5. 另外:之前 CRM 小幫手曾答錯說「同仁不能手動綁定」,那是錯的——已驗證表單欄位和 API 都存在,同仁可以手動綁。

### MAN-客戶-05 客戶詳細頁能看到什麼?(消費紀錄、訂單、來源標籤)(v1)

### MAN-客戶-06 旅客資料(護照拼音、素食、生日)要去哪裡改?(v1)
*頁面:瀏覽器網址列直接輸入 /passengers(側欄無入口)→ 旅客管理列表 → 該旅客「編輯」*

1. 重點先講:訂單詳情頁(/orders/訂單編號)雖然有「旅客資料」區塊,會顯示每位旅客的證件號碼、出生日期、電話、素食類型、分房等,但那裡是「純顯示」,沒有任何編輯按鈕,不能在訂單頁改。

4. 列表顯示:姓名、性別、護照號碼、素食類型、特殊需求。找到該位旅客,按右邊「編輯」即可修改(見 MAN-客戶-07)。

### MAN-客戶-07 新增/編輯旅客有哪些欄位?素食類型怎麼填?(v1)
*頁面:/passengers → 「+ 新增旅客」或該列「編輯」→ 填表 → 「儲存」*

1. 在旅客管理頁(/passengers)按右上角藍色「+ 新增旅客」,或在列表按某位旅客的「編輯」,都會進到旅客表單。
2. 必填欄位:「訂單」(下拉選單選這位旅客屬於哪張訂單,選項顯示的是訂單編號——旅客一定掛在訂單底下,不能獨立存在)和「姓名」。
3. 其他欄位:性別(男/女/其他)、出生日期、國籍、護照號碼、素食類型、特殊需求、緊急聯絡人、緊急電話。
4. 素食類型下拉有四個選項:無 / 素食 / 全素 / 佛教素食。
5. 填完按「儲存」,會回到旅客列表;「取消」放棄。
6. 小提醒:客人自己從 LINE 表單填的素食類型是中文字(如「全素」「蛋奶素」),後台這張表單存的是另一套英文代碼,所以列表上的「素食類型」欄可能看到中英文混雜(例如 vegan 或 全素),這是系統已知的雙標準現象,看到不一致不用驚慌,以實際文字意思為準。

### MAN-客戶-08 旅客資料是怎麼進到系統的?(LIFF 表單等資料來源)(v1)
*頁面:客人端:LINE 內開啟 LIFF 旅客資料表單 (/liff/traveler-form);後台修改:/passengers*

旅客資料主要是「客人自己在 LINE 那端填進來的」,同仁不用手動逐筆輸入。三個來源:
1. LIFF 旅客資料表單(最主要):客人在 LINE 裡點開我們發的連結,會打開「旅客資料表單」網頁(/liff/traveler-form),逐位填寫姓名、性別、證件號碼、出生日期、電話、素食類型(無/全素/蛋素/奶素/蛋奶素/五辛素)、飲食禁忌、緊急聯絡人,還能選房型分房(單人房/雙人房/三人房/四人房/三人一房(加床不加價))。送出後資料直接寫進系統的旅客資料(走 /liff/save-travelers 或 save-single-traveler)。
2. LINE 對話流程:客人在聊天中依引導逐步回覆旅客資料,系統會自動解析存檔(traveler-data 流程)。
3. 護照上傳/OCR:客人上傳護照照片,系統自動辨識寫入旅客與護照資料(/liff/save-passport、護照 OCR)。
另外 LIFF 還有其他客人端頁面:護照上傳(/liff/passport-form)、我的訂單(/liff/my-orders)、付款截圖上傳(/liff/payment-upload)、線上刷卡(/liff/payment-gateway)、行程確認(/liff/trip-confirm)——這些都是客人在 LINE 裡看到的畫面,不在後台選單裡。
客人填錯怎麼辦?同仁可在後台 /passengers 旅客管理頁修改(見 MAN-客戶-06,注意該頁目前可能查不到資料),或請客人重開 LIFF 表單自行更正。

### MAN-對話-01 進入 LINE 客服聊天頁與版面導覽(v1)
*頁面:側邊欄「LINE 客服」→ 聊天頁(/chat)*

1. 登入 CRM 後台,點左側選單的「LINE 客服」(MESSAGES),就會進入聊天頁(網址是 /chat)。

### MAN-對話-02 切換不同的 LINE 官方帳號收發訊息(v1)

### MAN-對話-03 回覆客人訊息與「訊息發送失敗」的判讀(v1)
*頁面:聊天頁(/chat)→ 點聊天室 → 最下方「輸入訊息...」輸入框*

1. 點進聊天室後,在畫面最下方的輸入框(提示文字「輸入訊息...」)打字,按 Enter(或送出鍵)送出,訊息就會經由 LINE 發給客人。
2. 如果 LINE 那邊發送失敗,畫面會跳紅色提醒「訊息發送失敗」,而且那則訊息左邊會出現紅色驚嘆號(!)。
3. 滑鼠移到紅色驚嘆號上,會顯示「訊息未送達」和具體原因。
4. 若原因是額度或權限問題(例如官方帳號免費訊息額度用完、被客人封鎖),提示會註明「請至 LINE Official Account Manager 升級方案或加值後再試」——這要去 LINE 官方後台處理,CRM 這邊無法解。
5. 如果跳的是「無法連線到伺服器,請稍後再試」,代表是系統連線問題,稍等再送一次,持續發生請找管理員。
6. 補充:發送走「先嘗試回覆、不行就改推播」的機制,失敗紀錄會留在對話裡,所以紅色驚嘆號不會憑空消失,重新發一則成功的即可。

### MAN-對話-04 傳圖片或檔案給客人/下載客人傳來的檔案(v1)

## 帳權

### MAN-帳權-01 怎麼登入 CRM 後台(用 LINE 登入,沒有帳號密碼)(v1)
*頁面:CRM 後台網址 → /login 登入頁 → 「使用 LINE 登入」按鈕 → LINE 授權畫面 → 自動回到後台首頁*

CRM 完全沒有帳號密碼,只能用你自己的 LINE 帳號登入。

步驟:
1. 打開 CRM 後台網址,沒登入會自動跳到登入頁(/login),畫面上有「偷懶辦公室 / Line CRM」標題。
2. 頁面提示「請使用 LINE 帳號登入」,按下唯一的綠色大按鈕「使用 LINE 登入」。
3. 會跳轉到 LINE 的授權畫面,用你手機上平常用的 LINE 帳號同意授權。
4. 授權成功後系統自動發一張通行證(JWT)並把你帶回後台首頁,不需要再輸入任何東西。

注意:
- 第一次登入系統會自動幫你建一個「唯讀(readonly)」身分的使用者,不用申請開通,但預設權限最低(見 MAN-帳權-04)。
- 不要去問管理員要密碼——系統根本沒有密碼這個東西。

### MAN-帳權-02 LINE 登入失敗或畫面顯示「登入失敗」怎麼辦(v1)
*頁面:/auth-callback 錯誤畫面(3 秒自動跳回)→ /login 登入頁(紅色「登入失敗」警示框)→ 再按「使用 LINE 登入」*

登入失敗時系統會把你導回來並顯示錯誤,絕大多數重試一次就好。

會看到的畫面:
1. 授權後的轉跳頁如果失敗,會顯示警告圖示和錯誤訊息(例如「登入失敗,請重新嘗試」或「登入過程發生錯誤,請重新嘗試」),並提示「即將重新導向至登入頁面...」,約 3 秒後自動回到登入頁。
2. 登入頁本身也可能在上方出現紅色的「登入失敗」警示框,內容是失敗原因。

處理方式:
1. 回到登入頁後,直接再按一次「使用 LINE 登入」重試。最常見的原因是授權碼過期或被重複使用(例如停在授權頁太久、或按了上一頁再試),重走一次流程即可。
2. 如果連續多次都失敗,把錯誤訊息截圖,回報系統管理員。

另外:如果你登入很久之後突然被要求重新登入,那是通行證(token)過期,系統會自動清掉過期通行證,屬正常現象,重新按「使用 LINE 登入」即可。

### MAN-帳權-03 怎麼登出 CRM 後台(v1)
*頁面:後台任一頁 → 左側選單最下方帳號區塊(阿玩旅遊 / 後台操作員)→ 登出圖示按鈕(提示文字「登出」)→ 回到 /login*

登出按鈕在左側選單最下面的帳號區塊。

步驟:
1. 看後台左側選單(側欄)最底部,有一塊帳號區(固定顯示「阿玩旅遊 / 後台操作員」,不會顯示你的名字,是正常的)。
2. 區塊右邊有一個登出圖示按鈕(滑鼠移上去會顯示「登出」)。
3. 按下去之後,系統會清掉這台電腦上的登入通行證,並把你帶回登入頁。

說明:
- 登出只是清除這台電腦的登入狀態,CRM 裡的客戶、訂單資料完全不受影響。
- 要換帳號:先登出,再用另一個 LINE 帳號按「使用 LINE 登入」即可。
- 在共用電腦上用完務必登出,否則下一個人打開瀏覽器就是你的身分。

### MAN-帳權-04 新同事第一次登入後是「唯讀」身分,要怎麼變成管理員(v1)
*頁面:升級路徑一:左側選單「設定」→「後台管理人員」(/settings/admin-members);升級路徑二:LINE 客服聊天頁 → 設定(齒輪)→「Line 帳號管理」→「成員管理」→「邀請管理員」*

任何人用 LINE 第一次登入,系統都會自動建立一個「唯讀(readonly)」身分,不會自動變成管理員。

要讓同事成為管理員,有兩條路:
1. 管理員直接加他:到左側選單「設定」區的「後台管理人員」頁,輸入他的 LINE User ID 新增(詳見 MAN-帳權-05)。
2. 發邀請連結給他:在聊天頁的「Line 帳號管理 → 成員管理」產生一次性邀請連結,請他點開接受(詳見 MAN-帳權-07、08)。

注意事項:
- 用「後台管理人員」這條路新增之前,對方必須先用 LINE 跟公司官方帳號互動過(例如加好友、傳過訊息),系統才有他的聯絡人紀錄,否則會顯示找不到。
- 已知系統怪癖:LINE 登入建立的使用者身分在後端固定是 readonly(程式裡註明「固定為 readonly」),而且登入發的通行證格式跟「需要管理員權限」的檢查對不起來(稽核報告已記載:真實 LINE 登入的通行證去打需要管理員權限的功能,理論上會被擋)。管理員名單實際記在另一張管理員成員表。若同事被加為管理員後仍遇到「需要管理員權限」的錯誤,請回報 Gary,不是你操作錯。

### MAN-帳權-05 在「後台管理人員」頁新增管理員(輸入 LINE User ID)(v1)
*頁面:左側選單「設定」→「後台管理人員」(/settings/admin-members)→「新增管理員」區塊 → 輸入 LINE User ID → 按「新增」*

步驟:
1. 左側選單往下找「設定」區塊,點「後台管理人員」(英文標 ADMINS),進入「後台管理人員」頁。
2. 在上方「新增管理員」區塊的輸入框,貼上對方的 LINE User ID——一串以大寫 U 開頭的代碼(輸入框提示「例:U1234567890abcdef...」;頁面也註明對方須已加入官方帳號好友)。
3. 按藍色「新增」按鈕,成功後對方會出現在下方「目前管理員」清單。

常見錯誤與處理:
- 顯示「找不到此 LINE User ID 的聯絡人」:代表對方還沒用 LINE 跟公司官方帳號互動過,請他先加官方帳號好友、傳一句訊息,再回來新增。
- 顯示「此用戶已經是管理員」:不用重複加,他已經在名單裡了。

### MAN-帳權-09 邀請連結顯示無效、過期或錯誤畫面怎麼辦(v1)
*頁面:重新產生:「LINE 客服」→ 設定(齒輪)→「Line 帳號管理」→「成員管理」→「邀請管理員」→「複製」新連結傳給同事(5 分鐘內完成)*

原因九成是「連結天生短命」:邀請連結是一次性、產生後只有 5 分鐘有效。超過 5 分鐘、或已經被成功接受過(按過「確認加入」),就會顯示「此邀請連結已過期」或「此連結已被使用」「此連結已過期 (時效 5 分鐘)」之類的訊息。和 Email 完全無關,系統用 LINE 登入,邀請不綁任何 Email。

處理步驟:
1. 請原本發連結的人回到「LINE 客服 → 設定(齒輪)→ Line 帳號管理 → 成員管理 → 邀請管理員」重新產生一條新連結。
2. 先和同事約好「我現在發,你馬上點」,讓他在 5 分鐘內完成:點連結 → 先用 LINE 登入 → 重開連結 → 按「確認加入」。
3. 說明:光「點開看」不會把連結用掉,連結是在按下「確認加入」成功的那一刻才標記已使用;真正的敵人是 5 分鐘時限,所以不要拖。

此功能目前異常(附帶說明):如果連結代碼根本不存在(例如複製時少貼了幾個字),系統不會好好顯示「找不到邀請」,而是回一個系統錯誤(500),「無效的邀請」卡片下面會出現一段看不懂的英文錯誤文字。遇到這種畫面,先檢查連結有沒有複製完整,再請對方重新發一條即可,不用驚慌。

### MAN-SYS-05 小幫手能做什麼、不能做什麼(能力邊界)(v1)
*頁面:CRM 後台任一頁 → 右下角「CRM 小幫手」*

1. 它「只教不做」:只能告訴你操作步驟,不能幫你按按鈕、建資料、查訂單、發訊息。這不是它客氣 — 程式層面就把它鎖成零工具(-t none 旗標寫死在橋接服務,呼叫端無法覆寫;已實證模型無法執行任何動作),它真的做不到。
2. 它不碰客人:小幫手是內部同仁專用,不會、也不能回覆客人的 LINE 訊息(人格鐵則:只教不做、不碰客人、不知道就說)。
3. 它的鐵則是「不知道就說」:不確定時會直接說不確定或請你回報管理員,而不是亂掰;遇到這種回答請照做,去找管理員。
4. 它的知識可能落後或有錯。如果它指的路在畫面上找不到,以實際畫面為準,並直接在對話裡跟它說「你說錯了」「沒有這個按鈕」 — 這類字眼會被固定規則自動抓去送審(見 MAN-SYS-07),就是正式的回報管道。

### MAN-SYS-06 小幫手的使用限制與速度(為什麼要等比較久)(v1)
*頁面:CRM 後台任一頁 → 右下角「CRM 小幫手」*

1. 字數:每則問題 1~500 字(輸入框打滿 500 字就打不進去了),問題請拆短、一次問一件事。
2. 頻率:每人每 60 秒最多 10 個問題,超過會出現「問太快了,請稍候再試」,等個一分鐘再問即可。
3. 速度:等 6~9 秒以上是「正常」的 — 每一問系統都會重新啟動一次 AI 程序(目前架構就是這樣設計,約 6-9 秒起跳),不是壞掉。前端最多等約 100 秒,超過就顯示逾時。
4. 回覆長度:最長 4000 字,再長會被裁掉;覺得沒講完就請它「繼續」。
5. 同一時間全公司最多兩個問題並行處理(程式用號誌鎖死),尖峰時段排隊會更久一點,屬正常現象。

### MAN-SYS-07 問小幫手的內容會被記錄嗎?它答錯了怎麼回報?(v1)
*頁面:CRM 後台任一頁 → 右下角「CRM 小幫手」(留痕與送審在背景自動進行)*

1. 會留紀錄:你問的每一句、它答的每一句(含失敗的)都寫進本地資料庫的 assistant_chat_logs 表,含發問者、訊息、回覆、狀態與回覆速度,供管理員觀測。所以:不要在小幫手視窗貼客人個資、密碼或金鑰。
2. 答錯的回報方式 = 直接在對話裡講。系統用固定規則(正規表達式比對,不是 AI 自己判斷)自動把兩類對話標記送審:
   - 小幫手的回答含「不確定/不知道/請回報給管理員」;
   - 你的訊息含質疑字眼(「做錯」「錯了」「不對吧」「沒有這個」「找不到」「不存在」「亂講」等)。
3. 送審排程每 5 分鐘掃一次,把待審對話推到 Gary 的 Telegram;Gary 拍板後會更新小幫手的知識庫(送審訊息即註明「拍板後請告知修正內容,將更新知識庫並 commit+push」),之後同樣的問題就會答對。
4. 原則是「寧可多送不可漏送」(程式註解白紙黑字),所以覺得它怪就大方質疑,不用客氣。

### MAN-SYS-08 小幫手出現錯誤訊息怎麼辦(訊息對照表)(v1)
*頁面:CRM 後台任一頁 → 右下角「CRM 小幫手」視窗內的錯誤訊息*

看到的訊息對照處理方式:
1. 「小幫手目前停用中」→ 管理員把總開關關了(一鍵停用機制),不是故障;等通知重新開放即可。

### MAN-SYS-09 儀表板的營收統計數字怪怪的(此功能目前異常)(v1)

### MAN-ORD-02 手動建立一筆新訂單(v2)
*頁面:訂單管理 /orders → 右上「+ 新增訂單」→ /orders/new → 填表 →「建立訂單」*

1. 進「訂單管理」列表頁,按右上角紫色「+ 新增訂單」按鈕,進到「新增訂單」表單(/orders/new)。
2. 必填欄位(打 * 的):
   - 訂單編號(例:252191)
   - 訂單日期(預設帶今天)
   - 客戶:兩種方式擇一——
     a. 「選現有客戶」:在搜尋框輸入姓名/電話/Email/LINE 名稱,系統即時從全部客戶搜尋,再從下拉選單點選那位客戶。
     b. 「新客戶」:按客戶欄右上的「新客戶」切換,直接輸入新客戶姓名(電話建議填)。送出時系統會先比對電話/姓名:已有這位客戶就直接綁定,沒有才自動建新客戶檔,不用先去客戶管理建檔。
   - 總金額 (NT$)
   - 報名人數(預設 1)
3. 選填欄位:團體行程(下拉選團,選了會自動帶入商品名稱)、商品名稱、訂金狀態與尾款狀態(未繳/部分繳納/已繳清/免繳,預設未繳)、素食類型(無/全素/蛋奶素/五辛素/海鮮素,預設無)、取貨方式(例:郵寄、自取)、備註。
4. 填好按下方紫色「建立訂單」,成功後會自動跳回訂單列表。
注意:
- 「新客戶」模式只填姓名沒填電話、而系統裡已有多位同名客戶時,會被擋下並要求補電話,避免訂單綁錯人。
- 編輯既有訂單時沒有「新客戶」切換,只能搜尋選取既有客戶。

### MAN-ORD-03 取消整筆訂單(重要:官網 WooCommerce 不會自動跟著取消)(v1)
*頁面:訂單管理 → 點進訂單詳情 /orders/:id → 右上紅色「取消訂單」→ 確認視窗 → 確定;之後手動去官網 WooCommerce 後台再取消一次*

1. 從訂單列表點進該筆訂單的詳情頁(/orders/:id),按右上角紅色「取消訂單」按鈕。
2. 會跳出確認視窗,確認後系統把訂單狀態改成「已取消」。訂單資料會保留,只是客人之後無法在 LINE 上對這筆訂單做任何操作。
3. 如果這筆是「阿玩旅遊訂購系統」來的夥伴訂單(有 external_id),系統會先通知夥伴系統釋放出發日名額;若夥伴系統那邊失敗,CRM 這邊也不會取消(會跳錯誤訊息),避免兩邊不一致。

【最重要】CRM 取消「不會」回寫官網 WooCommerce!
程式裡雖然有回寫官網的程式碼,但正式環境的 WooCommerce API 金鑰沒有設定(2026-06-10 稽核確認三個變數皆未設),所以這段實際上不會執行。客人如果是在官網下單的,你在 CRM 取消之後,**一定要自己再去官網 WooCommerce 後台把那筆訂單也取消一次**,不然官網會一直顯示訂單有效。
(反過來,在官網後台取消是會自動同步進 CRM 的,見 MAN-ORD-07。)

### MAN-ORD-04 部分取消人數(一單多人,只有幾位不去)(v1)
*頁面:訂單詳情 /orders/:id → 右上橘色「部分取消人數」→ 依序輸入人數/原因/旅客序號 → 確認*

1. 點進該筆訂單詳情頁,按右上角橘色「部分取消人數」按鈕(在紅色「取消訂單」按鈕旁邊)。
2. 系統會依序跳出輸入視窗:
   - 輸入要取消的人數(必須大於 0,且小於目前訂單人數;不能等於全員,全員不去請用「取消訂單」)
   - 輸入部分取消原因(可留空)
   - 如果訂單已有旅客名單,可選填要移除哪幾位旅客的序號(用逗號分隔,例如 1,3;數量必須等於取消人數;留空則只扣人數、不刪旅客資料)
3. 最後確認後,訂單會保持有效(active),只把報名人數扣掉、釋放相應名額;夥伴訂單(有 external_id)也會通知對方系統釋放指定名額,對方失敗則整個動作不執行。系統並會自動留一筆取消稽核紀錄。
注意:訂單只有 1 位旅客時這顆按鈕會變淡不能按,系統會要求改用全單取消。

### MAN-ORD-05 搜尋與篩選訂單(注意:頂部全域搜尋框沒有作用)(v1)
*頁面:訂單管理 /orders → 頁內搜尋框 + 「全部訂單」「全部付款」兩個下拉 + 表頭排序 + 「重置」*

1. 請用「訂單管理」頁面裡的搜尋框(提示文字是「搜尋訂單號、商品名、客戶...」),輸入訂單號、商品名稱或客戶姓名/電話,停一下系統就會自動搜尋(搜的是全部訂單,不只當頁)。
2. 搜尋框右邊有兩個下拉篩選:
   - 訂單狀態:全部訂單/有效訂單/已取消
   - 付款狀態:全部付款/訂金未繳/部分繳納/已繳清/免繳
3. 點表頭的「訂單號」「日期」「建立」「金額」可以切換排序(再點一次反向)。

【附帶提醒】只有「搜尋框」是搜全部訂單;「訂單狀態」「付款狀態」兩個下拉篩選和表頭排序都只作用在目前畫面這一頁的 50 筆,翻頁要重看。要篩的對象很多時,建議先用搜尋縮小範圍再篩。

### MAN-ORD-06 看懂訂單的各種狀態標籤(v1)
*頁面:訂單管理列表的「訂金」「尾款」欄 + 訂單詳情頁;修改走 訂單編輯 /orders/:id/edit 的兩個狀態下拉*

訂單有三組獨立的狀態,不要搞混:
1. 訂單本身的狀態(order_status):只有兩種——有效(active)和已取消(cancelled)。已取消的訂單在列表會整列變淡灰、掛紅色「已取消」標籤,而且不再顯示編輯鉛筆按鈕。
2. 訂金狀態:未付(紅)/部分(琥珀)/已付(綠)/免付(灰)/待確認(橘)。
3. 尾款狀態:同上五種,和訂金各自獨立。

在列表上,「訂金」「尾款」兩欄各有一個小圓標籤;在訂單詳情頁則顯示為「已付款/待確認/部分付款/未付款/免繳」。
要改訂金/尾款狀態:點該訂單的「編輯」,在編輯訂單表單裡用「訂金狀態」「尾款狀態」兩個下拉改(選項:未繳/部分繳納/已繳清/免繳),再按「更新訂單」。

小提醒:資料庫裡有少數舊訂單(2026-06-10 稽核時為 4 筆)的付款狀態是「pending(待確認)」,這個值不在正式選項白名單裡,這種單若直接更新付款狀態可能被系統擋下——遇到請回報管理員處理。

### MAN-ORD-07 官網 WooCommerce 訂單怎麼進 CRM(同步是單向的)(v1)
*頁面:無需操作:官網下單/改狀態會自動出現在 訂單管理 /orders;CRM 端修改不會回官網*

同步方向只有一個:官網 → CRM,不會反過來。
1. 客人在官網(WooCommerce)下單後,官網會自動把訂單推送(webhook)進 CRM:系統會自動建立或更新客戶資料和訂單,用官網訂單號去重,所以同一筆不會重複建。這條路 2026-06-10 有真實訂單(#256310)實際走通驗證過。
2. 官網那邊「取消、退款、失敗、丟到回收桶」的訂單,推送過來時 CRM 會自動把對應訂單標成「已取消」;官網直接刪除訂單也會讓 CRM 那筆變「已取消」。
3. 反方向不通:你在 CRM 改的任何東西(取消、改金額、改狀態)都**不會**回寫官網(正式環境 WooCommerce API 金鑰未設定),官網要改請直接去 WooCommerce 後台操作(詳見 MAN-ORD-03)。
4. 舊的 Gmail 信件匯入訂單機制已經退役(Google 授權早已過期),現在官網訂單全靠上面的自動推送,不需要人工做任何事;如果發現官網有單但 CRM 一直沒出現,請回報管理員檢查推送設定,不要自己手動重建,以免之後官網再推送造成重複。

### MAN-ORD-08 修改一筆既有訂單的資料(v1)
*頁面:訂單列表操作欄鉛筆圖示 或 訂單詳情右上「編輯」→ /orders/:id/edit →「更新訂單」*

1. 兩個入口都可以:
   - 訂單列表最右邊「操作」欄的鉛筆小圖示(編輯)
   - 或點進訂單詳情頁,按右上角紫色「編輯」按鈕
2. 都會進到「編輯訂單」表單(/orders/:id/edit),欄位和新增訂單一樣:訂單編號、訂單日期、客戶、團體行程、商品名稱、總金額、報名人數、訂金/尾款狀態、素食類型、取貨方式、備註。
3. 改完按紫色「更新訂單」儲存,會跳回訂單列表。

小提醒:在編輯表單改「報名人數」只是改數字,不會釋放夥伴系統名額、也不會留取消紀錄;如果是客人少幾位要退,請用詳情頁的「部分取消人數」(MAN-ORD-04),那才會正確釋放名額並留紀錄。

### MAN-ORD-09 訂單詳情頁還能做什麼(加價購/金流付款連結/旅客資料)(v1)
*頁面:訂單詳情 /orders/:id → 「加價購明細」區塊 / 「金流付款連結」區塊(作廢) / 「旅客資料」區塊(唯讀,要改去 /passengers 按編輯)*

點進一筆訂單(/orders/:id)後,除了取消/編輯,還有這些區塊:
1. 加價購明細:按「+ 新增」展開小表單,填項目名稱(如:加網卡、單房差)、金額(正數是加購、負數是扣款)、備註(選填),按「確認新增」;每個項目旁可「刪除」(會跳「確定要刪除此加價購項目?」確認)。下方會顯示加價購合計與應付尾款。已取消的訂單不能再加。
2. 金流付款連結:列出這筆訂單發出去過的付款連結(類型/平台/金額/狀態等)。狀態是「待付款」且未過期的,最右邊有「作廢」按鈕,按下會跳確認視窗(顯示金額與平台 TapPay/ECPay 綠界,並註明系統會即時向金流確認尚未收款)。如果客人其實已付款,系統會擋下並自動把狀態同步成已付款。

4. 行程名稱如果是藍色字,點了會跳到該團的團體行程頁;「客戶資訊」整張卡片點下去會跳到客戶資料頁。

## 護照

### MAN-PASSPORT-01 查某一團還有誰沒交護照:團體行程 → 證件管理分頁(v1)
*頁面:左側選單「團體行程」→ 點進該團 → 上方分頁「證件管理」*

重點先講:左側選單「沒有」護照管理這個項目(首頁的「證件管理」捷徑也被暫時隱藏了),護照進度是藏在「團體行程」裡面。

步驟:
1. 左側選單點「團體行程」(GROUP TOURS)。
2. 點進你要查的那一團,進入該團詳細頁。
3. 頁面上方有五個分頁:訂單總覽/證件管理/財務總覽/分房管理/領隊交辦單 → 點「證件管理」。
4. 最上方會顯示「護照上傳進度:X / Y 位旅客」(大陸團會顯示「證件齊全進度」,因為需要護照+台胞證兩樣)。
5. 下面的表格按訂單分組,欄位有:客戶/訂單/人數/護照進度(例如 2/3,全交齊會變綠色加勾勾)。
6. 點每列最左邊的三角形(▶)展開,就能看到這筆訂單每一位旅客的護照明細;「還沒交的旅客」會顯示橘色驚嘆號和「尚未上傳護照」字樣。

另外在「訂單總覽」分頁上方也有一張「護照」統計卡,顯示已交護照的訂單數/總訂單數,可以快速掃一眼。

注意:本機資料庫的護照與旅客資料表筆數可能偏少(2026-06-10 搬家體檢時這兩張表是空的,雲端備份內有 9 筆護照、51 筆旅客)。如果你點進去看到「尚無訂單」或進度全是 0,先確認該團底下是否真的有訂單與旅客,不一定是客人沒交;頁面顯示與按鈕功能本身是正常的。

### MAN-PASSPORT-02 幫客人手動補登/修改護照資料(後台代填)(v1)
*頁面:團體行程 → 該團詳細頁 → 「證件管理」分頁 → 展開訂單 → 旅客列右側「新增護照／編輯護照」按鈕*

可以,後台同仁可以直接幫旅客新增或修改護照資料,操作就在證件管理分頁裡:

1. 左側選單「團體行程」→ 點進該團 → 上方分頁「證件管理」。
2. 點訂單列最左邊的三角形(▶)展開旅客明細。
3. 每一位旅客名字右邊有一顆紫色邊框按鈕:還沒有護照的顯示「新增護照」,已有護照的顯示「編輯護照」。點下去會跳出視窗。
4. 視窗欄位:
   - 護照號碼(必填,會自動轉大寫)
   - 效期(必填,選日期)
   - 發照日期
   - 英文姓名(格式參考 SURNAME/GIVEN NAMES,自動轉大寫)
   - 國籍(下拉:台灣/中國/香港/澳門/日本/美國/其他)
   - 性別(男/女)
   - 出生日期
5. 按右下角紫色「儲存」。沒填護照號碼會提示「請輸入護照號碼」,沒選效期會提示「請選擇護照效期」。
6. 存好後畫面會自動重新整理,進度數字跟著更新。

小提醒:這個視窗是「補登文字資料」用的,不能在這裡上傳護照照片;照片要由客人從 LINE 的上傳護照流程拍照上傳(見 MAN-PASSPORT-03)。

### MAN-PASSPORT-03 客人怎麼從 LINE 上傳護照?(拍照自動辨識 OCR 流程)(v1)
*頁面:客人端:LINE「我的訂單」→ 訂單卡片「上傳護照」→ 點選旅客 → 拍攝或選擇護照照片 → 確認辨識結果 → 儲存護照資料*

客人是在 LINE 裡自己操作的,你可以照這個流程教客人:

1. 客人在 LINE 打開「我的訂單」頁,每張訂單卡片上有三顆按鈕:「填寫資料」「上傳護照」「繳費」→ 按「上傳護照」。
2. 進入護照上傳頁後,畫面提示「請點選旅客上傳護照資料,可拍照自動辨識」→ 點選要上傳的那一位旅客。
3. 點虛線框「拍攝或選擇護照照片」(可以現場拍,也可以從相簿選)。
4. 上傳後系統會顯示「正在辨識護照資料...」,自動辨識(OCR)完成後出現「辨識結果(請確認是否正確)」卡片,自動帶出:英文姓名/護照號碼/護照效期/國籍/出生日期/性別。
5. 辨識結果會自動填進下方欄位,客人確認或手動修正後按「儲存護照資料」。如果辨識失敗,畫面會提示「請手動填寫下方欄位」,照片照樣會留存。
6. 存完會自動跳到下一位還沒交的旅客;全部交完顯示「全部完成!所有旅客的護照資料已上傳完成」。

同仁端怎麼看結果:團體行程 → 該團 → 「證件管理」分頁,旅客明細會顯示護照號碼/國籍/效期和辨識狀態標籤(OCR 完成/待辨識/辨識失敗/手動輸入)。

技術備註(2026-06-10 體檢已驗證):辨識走雲端模型,本機 Ollama 目前從容器內連不到,系統會自動改走 OpenRouter,辨識功能本身正常可用。

### MAN-PASSPORT-04 催客人交護照:一鍵催整團/單筆訂單催繳(v1)
*頁面:團體行程 → 該團詳細頁 → 「訂單總覽」分頁 → 「護照」統計卡「一鍵催護照」;或訂單列「通知 ▼」→「催繳護照」*

有兩種催法,都在「團體行程 → 該團詳細頁 → 訂單總覽分頁」:

【整團一鍵催】
1. 「訂單總覽」分頁上方有四張統計卡:訂金/尾款/旅客資料/護照。
2. 「護照」卡片上如果還沒收齊,會出現「一鍵催護照」按鈕。
3. 按下後跳出「確認發送通知」視窗,寫明「確定要『催繳護照』?(僅通知未完成的客戶)」→ 按「確認發送」。
4. 只會發給「還沒交齊護照」的客戶,已交齊的不會被打擾。發完畫面會顯示「通知已發送:成功 X 筆,失敗 Y 筆」。

【單筆訂單催】
1. 同一個訂單列表裡,每筆訂單最右邊有「通知 ▼」按鈕。
2. 點開選單,選「催繳護照」(選單裡還有催繳訂金/催繳尾款/催填旅客資料)。
3. 一樣會跳確認視窗,標題會帶客戶名字(例如「催繳護照 — 王小明」),確認後只發給這一位。

注意:通知是透過 LINE 發出去的,客人要有綁定 LINE 才收得到(訂單列表的 LINE 欄綠勾=已綁定、灰叉=未綁定,沒綁定的會算在「失敗」筆數裡)。

### MAN-PASSPORT-05 看護照掃描檔與辨識狀態標籤的意思(v1)
*頁面:團體行程 → 該團詳細頁 → 「證件管理」分頁 → 展開訂單 → 旅客護照區塊「檢視掃描檔」/「辨識」標籤*

看照片:
1. 團體行程 → 該團 → 「證件管理」分頁 → 展開訂單(▶)。
2. 每位旅客的護照區塊裡,如果有上傳照片,右側會有藍色「檢視掃描檔」連結,點了會放大顯示原始照片(點黑色背景或 ✕ 關閉)。沒有這個連結=客人只有填文字資料、沒有照片(例如同仁後台代填的就不會有照片)。

辨識狀態標籤(在護照號碼旁邊的「辨識」欄):
- 「OCR 完成」(綠色):系統自動辨識成功,資料是機器讀出來的,建議肉眼跟掃描檔核對一次。
- 「待辨識」(黃色):照片已上傳但還沒辨識完成。
- 「辨識失敗」(紅色):機器讀不出來,資料是客人手動填的,務必對照掃描檔檢查。
- 「手動輸入」(藍色):資料是人工輸入的,沒有經過自動辨識。

核對重點:英文姓名拼音、護照號碼、效期,這三項出錯最容易影響開票。

### MAN-PASSPORT-06 護照效期不足的警示怎麼看(6 個月規則)(v1)
*頁面:團體行程 → 該團詳細頁 → 「證件管理」分頁 → 旅客護照明細的「效期」欄紅字「即將到期」*

系統兩邊都有提醒,但都「只提醒、不會擋件」,最後還是要靠同仁判斷:

【後台(同仁看的)】
- 在「證件管理」分頁的旅客護照明細裡,如果護照效期在「未來 180 天內」就到期,效期日期會變成紅字,下面多一行紅色小字「即將到期」。
- 看到紅字就要主動聯絡客人確認是否換發護照,不要等到開票才發現。

【客人端(LINE 上傳時)】
- 客人在 LINE 上傳護照時,如果效期距離出發日不足 6 個月,畫面會出現黃色警告卡「護照效期不足」,提醒「部分國家可能拒絕入境,建議確認護照效期是否符合目的地要求」。
- 這只是警告,客人還是可以照常按「儲存護照資料」送出,所以後台看到紅字仍要人工追蹤。

### MAN-PASSPORT-07 大陸團的台胞證:證件齊全=護照+台胞證(v1)
*頁面:團體行程 → 該團詳細頁 → 「證件管理」分頁(大陸團顯示「證件齊全進度(需護照+台胞證)」)*

大陸行程的團,系統會自動把證件要求變成「護照+台胞證」兩樣都要:

【後台怎麼看】
1. 團體行程 → 該團 → 「證件管理」分頁。
2. 大陸團的進度標題會顯示「證件齊全進度:X / Y 位旅客」,旁邊註明「(需護照+台胞證)」;表格欄位也會從「護照進度」變成「證件進度」。
3. 展開旅客明細,每位旅客會有兩個區塊:📘 護照(護照號碼/國籍/效期)和 📗 台胞證(台胞證號碼/發證日/效期),各自有「檢視掃描檔」。兩樣都有才算這位旅客完成;缺任一樣會顯示橘色驚嘆號「證件尚未齊全」和「尚未上傳台胞證」等字樣。

【客人端流程】
- 客人在 LINE 上傳時是兩步驟:先傳護照、存完自動接著傳台胞證(按鈕會變成「儲存台胞證資料」)。如果客人之前已交護照,下次點進來會直接跳到台胞證那一步。
- 台胞證欄位:台胞證號碼(必填,6-12 碼英數字)/發證日期/台胞證效期(必填)/照片(必拍)。
- 注意:自動辨識(OCR)目前「只支援護照」,台胞證要客人照著證件手動填,所以台胞證資料更需要同仁對照掃描檔核對。

【後台代填限制】後台「新增護照/編輯護照」視窗只能填護照欄位,沒有台胞證欄位;台胞證目前只能由客人從 LINE 上傳。

### MAN-PASSPORT-08 旅客資料頁(/passengers)改護照號碼,與隱藏的舊證件頁注意事項(v1)
*頁面:網址列直接輸入 /passengers(旅客管理)或 /passports(舊證件頁,不建議);側欄與首頁皆無入口*

系統裡還有兩個跟證件相關的「隱藏頁面」,左側選單和首頁捷徑都沒有入口(首頁的「旅客管理」「證件管理」捷徑在程式裡被註解隱藏),只能在網址列直接輸入路徑:

【旅客資料頁 /passengers — 可以用,但陽春】
1. 網址列輸入 /passengers,進入「旅客管理」頁,列表欄位:姓名/性別/護照號碼/素食類型/特殊需求。
2. 點該旅客的「編輯」,表單裡有「護照號碼」「國籍」「出生日期」等欄位,改完按「儲存」。
3. 注意:這裡改的是旅客(passengers)資料表上的護照號碼欄位,「不會」上傳照片、也不會動到證件管理分頁裡的護照檔案;日常作業建議優先用「團體行程 → 證件管理分頁」的編輯護照(MAN-PASSPORT-02),資料才完整。

【舊證件頁 /passports — 不建議使用】
網址列輸入 /passports 會出現一個舊版「證件管理」列表(有「+ 新增護照」按鈕)。這是早期留下的頁面,顯示的「已驗證/領隊保管中」狀態欄位與現行資料格式對不上,也沒有照片與 OCR 資訊;手冊立場:此頁面屬於舊功能,請一律改用「團體行程 → 證件管理分頁」。

【資料現況提醒】2026-06-10 搬家體檢發現本機資料庫的旅客/護照兩張表筆數偏少(當時為 0 筆,雲端備份有資料),所以這兩頁打開可能是空白或少量列表——看到空的先別慌,先確認資料是否已匯入,不是你操作錯誤。另外後端「待辨識護照清單」API(GET /api/v1/passports/pending-ocr)原本因資料表缺欄位回 500,已於 2026-06-11 第二批修復(改以 created_at 排序),現況正常;不過前端目前沒有任何頁面用到它,不影響日常操作。

### MAN-金流-01 客人匯款後,怎麼在付款管理確認收款(單筆)(v1)
*頁面:左側選單「付款管理」(PAYMENTS) → /payments → 切「待驗證」頁籤 → 點訂單展開 → 確認 / 金額有誤 / 退款*

1. 左側選單點「付款管理」(PAYMENTS),進入付款管理頁。
2. 上方有狀態頁籤:全部 / 待驗證 / 已確認 / 金額有誤 / 已退款 / 繳費連結。要對帳就切到「待驗證」。
3. 列表是按訂單分組的,點訂單那一列會展開底下的付款明細(類型、金額、匯款銀行、帳號後五碼、截圖、日期、備註、狀態)。
4. 待驗證的那筆,操作欄有三顆按鈕:
   - 綠色「確認」:跳出「確認付款成功」視窗,按「確認付款」即標記為已確認。
   - 橘色「金額有誤」:直接跳出輸入原因的視窗(「請輸入原因...」),填原因後按「標記金額有誤」,系統會通知客戶重新確認。
   - 紅色「退款」:同樣跳出輸入原因的視窗,填原因後按「確認退款」,標記為已退款(實際退錢要另外走後續流程)。
5. 按確認後系統會自動發 LINE 通知客人(收款確認+證件繳交通知);標金額有誤會發付款異常通知、退款會發退款通知,都是自動的。
注意:如果同一張訂單有兩筆以上待驗證,單筆按鈕會被收起來、顯示「由訂單統一操作」,要用訂單列上的「全部確認 / 全部駁回」(見 MAN-金流-02)。

### MAN-金流-02 一張訂單好幾筆付款,怎麼一次全部確認或全部駁回(v1)
*頁面:左側選單「付款管理」→ /payments → 訂單列右側「全部確認 (N 筆)」/「全部駁回 (N 筆)」*

1. 在「付款管理」頁,如果同一張訂單有 2 筆以上「待驗證」的付款,展開後每筆的單獨按鈕會被收起,只顯示「由訂單統一操作」。
2. 這時請看訂單那一列的右邊,會出現兩顆按鈕:
   - 綠色「全部確認 (N 筆)」:跳出「批量確認付款」視窗,顯示客戶、訂單編號、筆數和總金額,按「全部確認」一次全部標為已確認。
   - 橘色「全部駁回 (N 筆)」:跳出輸入原因的視窗,填原因後按「全部駁回」,整批標為「金額有誤」。
3. 訂單列右邊也會顯示「N 筆待驗證」黃色標籤、付款筆數與總額,方便快速掃。
4. 批量確認後系統一樣會自動發 LINE 通知客人。

### MAN-金流-03 客人的匯款截圖在哪裡看?截圖是怎麼進系統的?(v1)
*頁面:左側選單「付款管理」→ /payments → 展開訂單 → 「截圖」欄縮圖 → 點擊查看大圖*

1. 到「付款管理」頁,展開訂單後,每筆付款明細有一欄「截圖」,顯示小縮圖。
2. 滑鼠移上去會提示「點擊查看大圖」,點縮圖就會跳出大圖視窗,點右上角 X 或點黑色背景關閉。
3. 如果那一格是灰色的相機圖示,代表「無截圖」——這筆付款客人沒有附截圖,點了不會有反應。沒截圖的話可以改看旁邊的「匯款銀行」和「帳號後五碼」欄位來對帳(客人沒傳截圖時,系統會強制他填帳號後五碼,提示文字是「未上傳截圖時,請輸入帳號後五碼(5 位數字)」)。
4. 截圖的來源:是客人自己在 LINE 裡開「付款上傳」頁面(LIFF 表單)上傳的,上傳後這筆付款會以「待驗證」狀態建立,客人的 LINE 聊天裡也會出現「付款資訊已上傳」的訊息;同仁這邊沒有替客人上傳截圖的地方(後台手動補登付款的表單也沒有截圖欄位)。

### MAN-金流-04 金流訂單頁(線上刷卡連結)怎麼查、怎麼篩選(v1)

2. 上方篩選列有四個工具:
   - 狀態下拉:待付款 / 已付款 / 已作廢 / 已過期 / 失敗 / 全部(預設只顯示「待付款」)。

### MAN-金流-05 繳費連結金額打錯了,怎麼作廢?客人已經刷了會怎樣?(v1)
*頁面:左側選單「金流訂單」→ 該筆右側「作廢」;或 訂單詳情頁 → 「金流付款連結」卡片 → 「作廢」*

可以作廢,有兩個入口,效果一樣:

入口 A — 金流訂單頁:
1. 左側選單「金流訂單」→ 找到那筆連結(狀態必須還是「待付款」且未過期,才會有按鈕)。
2. 按該列右側紅框的「作廢」按鈕。

入口 B — 訂單詳情頁:
1. 進該張訂單的詳情頁(/orders/訂單id),往下找「金流付款連結」卡片。
2. 待付款的那筆右側同樣有紅框「作廢」按鈕。

按下後會跳確認視窗,顯示金額、平台(金流訂單頁的視窗還會多顯示訂單編號),並註明「系統會即時向金流確認尚未收款」。確認後:
- 成功:跳出「已作廢」(訂單詳情頁版本顯示「繳費連結已作廢」),狀態變成「已作廢」,且系統會自動發 LINE 通知客人連結已作廢。
- 如果客人其實已經刷卡付掉了:系統會擋下作廢,跳出「客戶已完成付款,無法作廢。系統已自動同步狀態為已付款。」——錢不會不見,狀態會自動修正成已付款。
- 如果金流端查詢失敗:會顯示「金流查詢失敗,請稍後再試」,稍後重按即可。

作廢後如果要重新收款,連結本身不能改金額,要請客人在 LINE 端重新走付款流程產生新連結(見 MAN-金流-08)。

### MAN-金流-06 付款管理頁的「繳費連結」分頁載不出資料 — 此功能目前異常,請改用金流訂單頁(v1)
*頁面:左側選單「付款管理」→ /payments → 「繳費連結」頁籤(異常,勿用) → 改用「金流訂單」/payment-links*

「付款管理」頁最右邊有一個「繳費連結」頁籤,但【此功能目前異常】:它呼叫的後端清單 API(GET /api/v1/payments/links)因路由順序問題(/links 被前面的動態路由攔截)從寫出來那天起就一直回錯誤(422),所以這個分頁永遠載不到資料,看起來像「尚無繳費連結」——不代表真的沒有連結。

正確做法:要查繳費連結,一律改用左側選單的「金流訂單」頁(/payment-links),那邊走的是另一條正常的 API(/payment-gateway/admin/links),資料完整、可篩選、也能作廢(見 MAN-金流-04、MAN-金流-05)。單一訂單的連結也可以在訂單詳情頁的「金流付款連結」卡片看到。

### MAN-金流-07 客人付現金或臨櫃匯款,怎麼手動補登一筆付款紀錄(v1)
*頁面:直接輸入網址 /payments/new(左側選單與付款管理頁皆無新增按鈕)*

1. 「付款管理」列表頁目前【沒有】「新增付款」按鈕,要手動補登請直接在瀏覽器網址列輸入:CRM 網址 + /payments/new(此頁面確實存在,標題「新增付款」,頂部有「返回付款列表」連結)。
2. 表單欄位(打 * 為必填):
   - 訂單 *:下拉選擇(顯示訂單編號與金額)。
   - 金額 *、付款日期 *(預設今天)。
   - 付款方式 *:現金 / 銀行轉帳 / 信用卡 / LINE Pay / 其他(預設銀行轉帳)。
   - 交易編號(選填)、備註(選填)。
3. 按「儲存」送出,完成後會自動跳回付款列表;按「取消」放棄。
4. 已存在的付款要修改,網址同理是 /payments/付款id/edit(頁面標題「編輯付款」,同一張表單)。
5. 提醒:手動補登的紀錄一樣會出現在付款管理列表;在還沒按「確認」之前,狀態欄會顯示「待驗證」,需要的話回列表按「確認」把它標成已確認(見 MAN-金流-01)。

### MAN-金流-08 繳費連結是誰產生的?同仁可以在後台幫客人開連結嗎?(v1)
*頁面:客人端:LINE 付款頁(LIFF)產生/重發;後台僅能查看/作廢:「金流訂單」與訂單詳情「金流付款連結」卡片;開關在「金流設定」*

1. 繳費連結(TapPay / ECPay 綠界的線上刷卡連結)是【客人自己在 LINE 端產生的】:客人在 LINE 開啟付款頁面(LIFF),選擇訂金或尾款後,系統產生連結並直接 LINE 推播給他;客人的付款頁面上也有「📨 獲取付款資訊(重發 LINE)」按鈕可以自己重發連結。
2. CRM 後台目前【沒有】任何「建立繳費連結」或「重發連結」的按鈕——整個前端管理介面找不到這個功能,同仁只能「查看」和「作廢」連結(金流訂單頁 / 訂單詳情頁)。
3. 所以客人說連結過期或被作廢時,正確回覆是:請客人回到 LINE 的付款頁面重新操作(或按頁面上的重發按鈕),系統會重新推播連結;不要在後台找開連結的按鈕,找不到的。
4. 客人會走線上金流還是傳統匯款截圖,取決於系統設定的付款模式(manual 手動匯款 / gateway 線上金流)與金流開關;金流平台的切換與「月上限」在左側選單「金流設定」(/settings/payment-gateway)管理——作用中金流達月上限時,新訂單的付款連結會無法產生。

### MAN-金流-09 付款管理出現紅色金額和警告三角形是什麼意思?(v1)
*頁面:左側選單「付款管理」→ /payments → 訂單列紅色金額+警告圖示 → 展開看「應收/已收」摘要*

1. 在「付款管理」頁,系統會自動比對每張訂單的「應收」和「已收」:
   - 訂金:把該訂單所有待驗證+已確認的訂金加總,跟應收訂金比。
   - 尾款:同樣邏輯跟應收尾款比。
2. 只要其中一項對不上(已收金額 > 0 但不等於應收),訂單列右邊的總金額會變【紅色】並出現警告三角形圖示——這就是「金額不符」的訊號。
3. 展開訂單後,明細表底下會有一行摘要:「應收訂金: NT$X / 已收: NT$Y」「應收尾款: NT$X / 已收: NT$Y」,對不上的那組會以紅字顯示,一眼就能看出差在哪裡(這行只在訂單有設定應收訂金時才會出現)。
4. 處理方式:先點截圖核對客人實際匯的金額,少匯就用「金額有誤」駁回並請客人補匯(系統會自動 LINE 通知客戶),多匯則依公司流程處理退差額。
