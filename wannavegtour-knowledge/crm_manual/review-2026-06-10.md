### MAN-催繳-01 怎麼進入「繳款提醒」頁面?誰能進?
- 路徑:直接在網址列輸入 /payment-reminders(需先登入;側欄會亮在「付款管理」(PAYMENTS),但側欄/首頁目前都沒有直接連結)
- 問法:催繳提醒要去哪裡看? / 繳款提醒在選單的哪裡? / 為什麼我在側欄找不到繳款提醒? / 首頁怎麼沒有繳款提醒的捷徑了?

1. 繳款提醒的頁面路徑是 /payment-reminders,頁面標題是「繳款提醒」;系統把它歸在左側選單「付款管理」(PAYMENTS) 底下(進到這頁時,側欄會亮在「付款管理」)。
2. 注意:側欄的「付款管理」按下去是進「付款管理」(/payments) 頁,側欄上沒有「繳款提醒」的直接連結;首頁的「繳款提醒」捷徑目前被暫時隱藏(程式碼註解掉了);整個前端也沒有其他按鈕連過去。所以目前唯一的進法是直接在瀏覽器網址列輸入 /payment-reminders。
3. 權限:要先登入 CRM 後台(用 LINE 登入)才能看,沒登入會被導回登入頁。登入之後就能進——程式碼上只檢查「有沒有登入」,沒有再分管理員或一般帳號。(以前流傳「僅管理員可進」,程式碼裡沒有這個限制。)
4. 沒有叫「催繳管理」的選單——以前小幫手講過這個名字,是錯的,不要去找。

### MAN-催繳-02 幫一筆訂單新增催繳提醒(欄位怎麼填)
- 路徑:繳款提醒列表 (/payment-reminders) → 右上「+ 新增提醒」→ 新增繳款提醒 (/payment-reminders/new)
- 問法:我要幫訂單建催繳提醒,要去哪裡按? / 新增繳款提醒哪些欄位必填? / 催繳提醒的提醒編號要怎麼寫? / 建提醒的時候訂單資料要自己打嗎?

1. 進「繳款提醒」列表頁 (/payment-reminders),按右上角藍色「+ 新增提醒」,進入「新增繳款提醒」頁 (/payment-reminders/new)。
2. 必填三項(沒填會跳「提醒編號、訂單和截止日期為必填」):
   - 提醒編號:自己編一個,例如 REM-2024-001。
   - 訂單:用下拉選(顯示「訂單編號 - 客戶姓名」),選好後「訂單編號」「客戶姓名」會自動帶入(新增時「訂單編號」是唯讀的,不用自己打)。
   - 截止日期:選日期後下方會即時顯示「已逾期 X 天」(紅字)、「今天到期!」或「X 天後到期」。
3. 其他欄位:
   - 提醒類型:定金 / 尾款(預設定金)。
   - 提醒狀態:待處理 / 已完成 / 已逾期 / 已取消(新建就留「待處理」)。
   - 提醒次數:預設 0,不用動。
   - 付款狀態:未繳 / 已繳 / 免繳(新建通常留「未繳」)。
   - 備註:選填。
4. 按「新增提醒」儲存,會跳回列表頁。注意:列表頁本身顯示有異常(見 MAN-催繳-03),跳回去看到空白或怪資料不代表沒建成功。
5. 注意:沒有「應繳金額」這種欄位,金額不在提醒表單上;建好提醒系統也不會自動發送,發送都要人工處理(見 MAN-催繳-04)。

### MAN-催繳-03 繳款提醒列表頁怎麼看?(此頁顯示目前異常)
- 路徑:繳款提醒列表 (/payment-reminders)
- 問法:繳款提醒列表的截止日期怎麼顯示怪怪的? / 列表的金額欄是什麼?為什麼頁面壞掉? / 繳款提醒列表狀態顏色代表什麼?

1. 列表表格設計上有六欄:訂單 / 類型 / 截止日期 / 金額 / 狀態 / 操作。狀態設計上分:待處理(黃)、已發送(藍)、已繳清(綠)、已取消(灰)。
2. 【此功能目前異常】列表程式讀的欄位名稱和後端送回來的資料對不上(程式讀 due_date / amount_due / status,但後端實際欄位是 deadline_date / reminder_status / payment_status,而且後端根本沒有 amount_due 金額欄位):
   - 「截止日期」會顯示成無效日期。
   - 「金額」欄讀不到資料會出錯——只要列表裡有任何一筆提醒,整頁幾乎一定當掉、顯示空白。
   - 「狀態」標籤顯示不出正確狀態。
   - 「訂單」欄會顯示一串訂單 ID 而不是訂單編號。
3. 替代作法:因為整頁可能空白,列表上的「編輯」按鈕也可能按不到。要確認某筆提醒的正確內容,可以直接在網址列輸入 /payment-reminders/{該筆ID}/edit 進編輯頁看(編輯頁欄位是正確的;ID 請管理員從後端查),或整筆都請管理員從後端 API 查。
4. 此頁修好前,請不要依列表上顯示的日期/金額/狀態做判斷。

### MAN-催繳-04 怎麼把催繳提醒發送給客人?(發送功能目前異常)
- 路徑:繳款提醒列表 (/payment-reminders) → 操作欄「發送」(目前按鈕不會出現,需管理員走後端 API)
- 問法:催繳提醒怎麼發給客人? / 為什麼列表上看不到發送按鈕? / 系統會自動發催繳訊息嗎? / 按發送之後客人 LINE 會收到什麼?

1. 先講重點:系統不會自動發送催繳訊息,所有發送都是人工觸發。
2. 設計上,列表「操作」欄在提醒是「待處理」時會出現「發送」按鈕,按了成功會跳「提醒已成功發送」。
3. 【此功能目前異常】兩層問題:
   - 因為列表欄位名稱不一致(見 MAN-催繳-03),列表頁本身可能整頁空白,而且「發送」按鈕的顯示條件永遠不成立,按鈕根本不會出現。
   - 就算出現,前端呼叫的 API 路徑(POST .../send)和後端實際路徑(PUT .../send-reminder)對不上,也會失敗。
4. 後端的發送 API(PUT /api/v1/payment-reminders/{id}/send-reminder)在 2026-06-10 已修復並通過測試:會依提醒類型發 LINE(定金=文字訊息、尾款=Flex 卡片);就算 LINE 發送失敗也不會報錯,仍會把提醒次數 +1 並記錄發送時間。
5. 但注意:修復時只驗證了「發送失敗時系統行為正確」,「真的把訊息送到客人 LINE」這條成功路徑還沒實測過——尤其定金文字提醒在修復前從來沒成功發出過。
6. 實務作法:需要發送催繳時,請找管理員從後端 API 操作,並先用測試帳號驗證一次訊息真的送得出去,再對真客人發。

### MAN-催繳-05 客人繳清了,怎麼把催繳提醒結掉?
- 路徑:繳款提醒列表 (/payment-reminders) → 該筆「編輯」→ 編輯繳款提醒 (/payment-reminders/{id}/edit) → 提醒狀態=已完成、付款狀態=已繳 → 「更新提醒」
- 問法:客人已經付錢了,提醒要怎麼標記完成? / 繳款提醒怎麼結案? / 列表上怎麼沒有完成按鈕?

1. 列表的操作欄只有「發送」和「編輯」兩個按鈕,沒有獨立的「完成」按鈕——要用「編輯」來結。
2. 步驟:
   - 在列表找到該筆,按綠色「編輯」,進「編輯繳款提醒」頁。(如果列表頁因顯示異常整頁空白——見 MAN-催繳-03——就直接在網址列輸入 /payment-reminders/{該筆ID}/edit,ID 請管理員從後端查。)
   - 「提醒狀態」下拉改成「已完成」。
   - 「付款狀態」下拉改成「已繳」(如果是公司同意不收的,選「免繳」)。
   - 按「更新提醒」儲存。
3. 編輯頁上「訂單」下拉在編輯模式是鎖住不能換的(提醒不能改掛到別的訂單);其他欄位都可改。
4. 給管理員參考:後端也有一鍵完成的 API(PUT /api/v1/payment-reminders/{id}/complete,把提醒狀態改為已完成)和標記已繳的 API(PUT /{id}/mark-paid),這幾支在 2026-06-10 已修復、測試通過。

### MAN-催繳-06 提醒建錯了,怎麼取消或刪掉?
- 路徑:繳款提醒列表 (/payment-reminders) → 該筆「編輯」→ 提醒狀態=已取消 → 「更新提醒」(徹底刪除需管理員走 API)

1. 建錯的提醒,標準作法是「取消」而不是刪除:
   - 在列表按該筆的「編輯」進編輯頁。(列表頁若因顯示異常整頁空白——見 MAN-催繳-03——就直接在網址列輸入 /payment-reminders/{該筆ID}/edit。)

   - 按「更新提醒」儲存。
2. 如果只是填新表單填到一半想放棄,直接按表單下方灰色「取消」按鈕回列表即可,不會存任何東西。
3. 要把整筆提醒從系統徹底刪掉:前端頁面上沒有刪除按鈕,要請管理員用後端 API 刪(DELETE /api/v1/payment-reminders/{id},已驗證可用)。一般情況用「已取消」就好,留紀錄比較安全。
4. 給管理員參考:後端也有一鍵取消 API(PUT /{id}/cancel),2026-06-10 已修復、測試通過。

### MAN-催繳-08 要作廢「繳費連結」不是在繳款提醒——在「金流訂單」
- 路徑:側欄「金流訂單」(CASH FLOW) → 金流訂單列表 (/payment-links) → 該筆連結列右側「作廢」按鈕(僅待付款且未過期的連結有) → 確認視窗
- 問法:客人金額打錯,發出去的繳費連結要去哪裡作廢? / 催繳提醒裡面怎麼沒有作廢按鈕? / 作廢繳費連結客人已經付款了會怎樣?

1. 常見搞混:「繳款提醒」(催繳)和「繳費連結」(金流收款連結)是兩個不同功能。繳款提醒頁面裡沒有任何作廢繳費連結的按鈕。
2. 要作廢繳費連結,去左側選單「金流訂單」(CASH FLOW),進「金流訂單」頁 (/payment-links):
   - 找到那筆連結,按該列最右邊紅框的「作廢」按鈕(按下後會顯示「處理中」)。注意:只有「待付款」且未過期的連結才會有這顆按鈕,其他狀態只會顯示「—」。
   - 會跳確認視窗,顯示「確定要作廢這筆繳費連結?」加上訂單編號、金額、平台(TapPay 或 ECPay 綠界),並註明「系統會即時向金流確認尚未收款」。
   - 確認後成功會跳「已作廢」。依程式碼,作廢成功後系統也會發 LINE 通知客人連結已作廢(這段發送只做過程式碼檢查,沒實測過)。
3. 如果客人其實已經刷卡付掉了:系統會擋下來並提示「客戶已完成付款,無法作廢。系統已自動同步狀態為已付款。」——不會把已付款的單作廢掉。
4. 如果跳「金流查詢失敗,請稍後再試。」,等一下再按一次即可。

## 分房(8 條)
### MAN-分房-01 分房功能在哪裡?入口總覽
- 路徑:側欄「分房審核」或首頁「分房變更審核」卡 → /room-changes;房型分配管理需直接輸入網址 /room-allocations
- 問法:分房要去哪裡用? / 側欄怎麼沒有房型分配? / 分房管理的選單在哪? / 怎麼進到房型分配那一頁?

分房相關有兩個頁面,入口不一樣,請注意:
1. 「分房審核」:左側選單就有,按側欄的「分房審核」(英文標 ROOMING、盾牌圖示),會進到「分房變更審核」頁(網址 /room-changes),這裡是審核客人送來的換房申請。首頁的管理模組區也有一張「分房變更審核」捷徑卡可以進同一頁。

### MAN-帳權-02 LINE 登入失敗或畫面顯示「登入失敗」怎麼辦

1. 請原本發連結的人回到「LINE 客服 → 設定(齒輪)→ Line 帳號管理 → 成員管理 → 邀請管理員」重新產生一條新連結。

### MAN-ORD-07 官網 WooCommerce 訂單怎麼進 CRM(同步是單向的)

### MAN-金流-08 繳費連結是誰產生的?同仁可以在後台幫客人開連結嗎?
- 路徑:客人端:LINE 付款頁(LIFF)產生/重發;後台僅能查看/作廢:「金流訂單」與訂單詳情「金流付款連結」卡片;開關在「金流設定」

### MAN-金流-09 付款管理出現紅色金額和警告三角形是什麼意思?
- 路徑:左側選單「付款管理」→ /payments → 訂單列紅色金額+警告圖示 → 展開看「應收/已收」摘要
- 問法:付款列表金額怎麼變紅色還有驚嘆號? / 應收和已收不一樣系統會提醒嗎? / 怎麼快速看出客人匯的錢跟應收對不上?

1. 在「付款管理」頁,系統會自動比對每張訂單的「應收」和「已收」:
   - 訂金:把該訂單所有待驗證+已確認的訂金加總,跟應收訂金比。
   - 尾款:同樣邏輯跟應收尾款比。
2. 只要其中一項對不上(已收金額 > 0 但不等於應收),訂單列右邊的總金額會變【紅色】並出現警告三角形圖示——這就是「金額不符」的訊號。
3. 展開訂單後,明細表底下會有一行摘要:「應收訂金: NT$X / 已收: NT$Y」「應收尾款: NT$X / 已收: NT$Y」,對不上的那組會以紅字顯示,一眼就能看出差在哪裡(這行只在訂單有設定應收訂金時才會出現)。
4. 處理方式:先點截圖核對客人實際匯的金額,少匯就用「金額有誤」駁回並請客人補匯(系統會自動 LINE 通知客戶),多匯則依公司流程處理退差額。
